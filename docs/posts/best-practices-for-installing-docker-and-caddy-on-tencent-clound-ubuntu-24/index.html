腾讯云 Ubuntu 24.04 安装 Docker 与 Caddy 最佳实践指南

2026-06-08 · 9 分钟 · 约 4440 字

本文通过 AI 生成，主要用我个人快速安装软件使用，并备注了 https 证书无法颁发的解决办法。

本指南专门针对腾讯云 Ubuntu 24.04 LTS (Noble Numbat) 环境编写，完美避开了国内网络访问 Docker 官方源被重置的痛点，采用国内大厂加速方案，并整合了 Caddy 自动化 HTTPS 服务的标准生产级部署流程。

一、 环境清理与准备#

为了防止 Ubuntu 系统自带的旧版本组件（或 Snap 包）与官方最新版本发生冲突，在安装前必须对环境进行清理。

1. 卸载冲突组件#

执行以下命令，安全移除系统中可能存在的旧版 Docker 核心及容器运行时：

sudo apt-get remove docker docker-engine docker.io containerd runc

2. 安装基础依赖包#

更新本地软件包索引，并安装支持 HTTPS 传输、证书管理及网络下载的基础工具：

sudo apt-get update
sudo apt-get install ca-certificates curl gnupg -y

二、 Docker 核心组件安装（腾讯云环境优化）#

由于直接连接 Docker 官方 APT 源经常遭遇 Connection reset by peer 错误，本步骤采用 阿里云镜像源 进行替换，以确保网络无缝连接。

1. 配置国内镜像源 GPG 密钥#

创建专门的密钥存放目录，并下载经过安全校验的 Docker 密钥：

sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://mirrors.aliyun.com/docker-ce/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

2. 写入 APT 软件源#

将软件源指向国内镜像站，同时系统会自动匹配 Noble (24.04) 版本的系统代号：

echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://mirrors.aliyun.com/docker-ce/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

3. 稳健安装 Docker 与 Compose#

再次更新索引并执行安装。此命令不仅会安装 Docker 引擎，还会默认安装最新的 Docker Compose V2 插件：

sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin -y

三、 生产环境最佳实践配置#

安装完成后，需要对 Docker 进行深度优化。以下配置能够实现非 root 用户免 sudo 维护、防止宿主机磁盘被日志撑爆、以及享受腾讯云内网级别的镜像下载加速。

1. 配置免 sudo 运行 Docker#

默认情况下，非 root 用户每次执行 docker 命令都必须加 sudo。通过将当前用户加入 docker 用户组，可以安全地免除 sudo：

# 创建 docker 用户组（通常安装时已自动创建）
sudo groupadd docker

# 将当前登录用户（$USER）加入组中
sudo usermod -aG docker $USER

核心提示：为了使此组策略修改生效，您需要执行 newgrp docker 命令，或者注销并重新建立 SSH 连接。

2. 高级全局配置文件优化 (daemon.json)#

创建或覆盖 Docker 的全局配置文件，集中解决镜像加速与日志暴涨问题：

sudo vim /etc/docker/daemon.json

在文件中粘贴以下标准 JSON 配置：

{
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com",
    "https://docker.m.daocloud.io"
  ],
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}

配置参数深度解析：#

  • registry-mirrors：首位配置了 mirror.ccs.tencentyun.com，这是腾讯云内网镜像加速器，只有云服务器在内网访问时生效，速度极快且不消耗公网流量；第二位配置了高可用的公网备用源。

  • log-driver 与 log-opts：将每个容器的日志上限死锁在 10MB，且最多只保留 3 个历史切片文件。彻底杜绝因业务容器疯狂输出日志导致系统盘突发写满、系统崩溃的隐患。

3. 应用配置并启动#

# 重新加载 systemd 守护进程
sudo systemctl daemon-reload

# 重启 Docker 服务以应用新配置
sudo systemctl restart docker

# 设置开机自启
sudo systemctl enable docker

四、 整合 Caddy 实现自动化 HTTPS 逆向代理#

Caddy 具备内置的 ACME 客户端，只要端口放行且域名解析正确，它就能完全自动地申请、安装、并续签由 Let’s Encrypt 或 ZeroSSL 颁发的免费 HTTPS 证书。

1. 前置条件检查#

  1. 安全组放行：必须登录腾讯云控制台，在服务器关联的“安全组”中添加入站规则，放行 80/TCP、443/TCP 以及 443/UDP（用于支持最新的 HTTP/3 协议）。
  2. DNS 域名解析：请确保您的主域名或二级域名已添加 A 记录，并精准指向该腾讯云服务器的公网 IP。

2. 创建独立部署工程#

在宿主机上建立标准的持久化目录架构：

sudo mkdir -p /opt/caddy/config
cd /opt/caddy/config

3. 编写 Caddyfile 配置文件#

sudo vim Caddyfile

键入以下规范内容（请根据注释修改为您真实的域名和邮箱）：

{
    # 强烈建议配置真实邮箱，当证书续签遇到罕见故障时，Let's Encrypt 会发送通知
    email your-email@example.com
}

# 替换为您解析好的真实域名
yourdomain.com {
    # 场景 A：反向代理到本地运行的伴生业务（如本地的 8080 端口）
    # 172.17.0.1 是 Docker 默认网桥的宿主机 IP
    reverse_proxy 172.17.0.1:8080

    # 场景 B：如果您目前没有后端业务，只想先测试 HTTPS 是否顺畅，可取消下行注释
    # respond "Hello, Caddy Secure HTTPS HTTPS/3 works perfectly!"
}

4. 编写编排文件 docker-compose.yml#

sudo vim docker-compose.yml

写入以下内容，确保数据卷（Volumes）将证书数据留存至宿主机：

version: "3.7"

services:
  caddy:
    image: caddy:2-alpine
    container_name: caddy
    restart: unless-stopped
    ports:
      - "80:80"
      - "443:443"
      - "443:443/udp" # 必须映射 UDP 端口以启用 HTTP/3 性能加速
    volumes:
      - ./config:/etc/caddy
      - caddy_data:/data      # 极其关键：Caddy 自动申请的证书、私钥存放于此，遗失会导致高频申请触发限流
      - caddy_config:/config  # 存放 Caddy 运行时的集群状态及临时配置
      - /var/www/html:/var/www/html

volumes:
  caddy_data:
    external: false
  caddy_config:
    external: false

5. 启动与验证#

在 /opt/caddy 目录下直接执行启动命令：

docker compose up -d

启动后，可以实时追踪容器日志，观察 Caddy 与证书签发机构的握手状态：

docker compose logs -f caddy

当日志中刷出 successfully activated certificate 及 acquiring lock 成功等字样时，即代表 HTTPS 证书已在后台秒级自动申请完毕。此时通过浏览器直接敲入 [https://yourdomain.com](https://yourdomain.com)，即可享受安全的绿锁加密连接。

5.1 证书颁发失败的解决方法#

若在日志出现了“https://acme.zerossl.com/v2/DV90”，Caddy 在向 Let’s Encrypt 申请失败或触发限制后，会自动 fallback（降级）到备用机构 ZeroSSL。

可以通过在 Caddyfile 中添加 cert_issuer acme 来解决。

{
	email your-email@example.com
	
	cert_issuer acme
}

# 替换为您解析好的真实域名
yourdomain.com {
    # 场景 A：反向代理到本地运行的伴生业务（如本地的 8080 端口）
    # 172.17.0.1 是 Docker 默认网桥的宿主机 IP
    reverse_proxy 172.17.0.1:8080

    # 场景 B：如果您目前没有后端业务，只想先测试 HTTPS 是否顺畅，可取消下行注释
    # respond "Hello, Caddy Secure HTTPS HTTPS/3 works perfectly!"
}
